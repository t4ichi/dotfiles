現在のブランチの変更内容を分析し、PRテンプレートに沿ったPull Requestを作成してください。

## 手順

1. 以下のコマンドを並列で実行し、現在の状態を把握する
   - `git status` で未コミットの変更を確認
   - `git branch --show-current` で現在のブランチ名を取得
   - `git log --oneline -20` で直近のコミット履歴を確認
   - ベースブランチ（main または develop）との差分: `git log --oneline main..HEAD` および `git diff main...HEAD --stat`

2. 未コミットの変更がある場合はユーザーに報告し、先にコミットするか確認する

3. リポジトリに `.github/PULL_REQUEST_TEMPLATE.md` が存在するか確認し、あればそのテンプレートを使用する。なければ下記のデフォルトテンプレートを使用する

4. ブランチ名・コミット履歴・diff から変更内容を分析し、PRのタイトルとボディを作成する

5. ユーザーに以下を確認する（1つの質問UIでまとめて聞く）
   - **ベースブランチ**: `main` または `develop` のどちらにマージするか
   - **PR種別**: 通常PR または draft PR

6. `gh pr create` でPRを作成する

## PRタイトルのルール

- 簡潔に変更内容を表す（70文字以内）
- リポジトリの慣例（日本語 or 英語）に合わせる

## PRボディのルール

テンプレートがある場合はその形式に従う。ない場合は以下のデフォルト形式を使う:

```markdown
## 概要
- 変更の目的と内容を1〜2行で

## 変更内容
- [x] 具体的な変更点をリストアップ

## 影響範囲
- 変更が影響するファイル・機能

## 確認事項
- [ ] Lint が PASS
- [ ] テストが PASS
```

## PRボディ作成時の注意

- テンプレートの記入例やプレースホルダ（例: `XXX API の〜`, `DEV-456`, `pages/speech/*`）はそのまま残さず、実際の変更内容に置き換える
- 記入不要なセクション（UI/動作イメージ等）は「なし」または該当なしと記載する
- `<!-- コメント -->` は削除する
- チェックリストは実際に確認した項目のみチェックを入れる

## gh pr create の実行形式

```bash
gh pr create --title "PRタイトル" --body "$(cat <<'EOF'
PRボディ
EOF
)"
```

## 注意事項

- `git push` はPR作成に必要な場合のみ実行する（ユーザーに確認してから）
- リモートにブランチがまだ push されていない場合は、push が必要な旨をユーザーに伝え確認する
- ベースブランチとdraft PRの選択はユーザーに必ず確認する
